## 4. 论文下载与本地标准化：这是必须执行的任务

你要在仓库中实现一个可重复执行的 paper ingestion pipeline。

### 4.1 先检查本地是否已有论文压缩包
在仓库和 `papers/raw_archives/` 下搜索：
- `.zip`
- `.tar`
- `.tar.gz`
- `.tgz`
- `.gz`

如果已经有下载好的论文压缩包，就直接处理，不要重复下载。

### 4.2 如果本地没有，就自动下载
从以下来源构建下载列表：
1. 官方 `references.py`（2025 + 2026 current）
2. 仓库中的 `cs336_paper_manifest.csv/json`（若存在）
3. 额外强制纳入的系统论文（即使不在 references.py 里也要收录）：
   - FlashAttention
   - FlashAttention-2
   - Triton

### 4.3 下载规则
对每篇论文：
1. **优先尝试下载 TeX / source**
   - 若是 arXiv，优先尝试 `https://arxiv.org/e-print/<arxiv_id>`
2. 如果没有公开 source：
   - 下载 PDF 或官方页面
   - 记录 `source_unavailable`
3. 所有原始下载统一进入：
   - `papers/raw_archives/`
   - `papers/metadata/`

### 4.4 处理规则（必须严格执行）
对所有已经下载的论文压缩包，执行：

1. **解压所有压缩包**
2. **解压成功后删除原始压缩包**
3. **把解压出来的原始文件夹重命名为论文标题**
   - 文件夹名必须合法
   - 做 slug/sanitize
   - 禁止含非法路径字符
4. 如果论文是多 TeX 文件结构：
   - 创建 `merged_paper.tex`
   - 递归展开 `\input{}` 和 `\include{}`
   - 尽量保持正文顺序
5. 在 `merged_paper.tex` 中：
   - 删除所有注释
   - 删除多余空行
   - 保留正文结构
6. 如果无法合并：
   - 记录失败原因到 `papers/metadata/source_failures.json`
   - 不要静默失败

### 4.5 生成 manifest
对每篇论文生成结构化元数据：
- title
- authors（若可得）
- year
- source_url
- local_folder
- source_type (`arxiv_source`, `pdf_only`, `html_only`, `manual`)
- merged_tex_path
- status (`ok`, `partial`, `failed`)
- cs336_2025 (bool)
- cs336_2026 (bool)
- topic_hint

写入：
- `papers/manifest.csv`
- `papers/manifest.json`

## 5. 教材写作风格：不可违反

### 5.1 面向对象
默认读者是：
- CS 本科生
- 会 Python
- 会一点 AI 调包 / 训练脚本 / vibe coding
- 但不一定懂：
  - NLP
  - language modeling 历史
  - linguistics
  - CUDA
  - distributed systems
  - scaling laws
  - alignment

### 5.2 写作必须同时满足两个条件
1. **数学严格**
2. **可读、可直觉理解**

所以每章都要有：
- precise definition
- intuition
- worked example
- implementation consequence
- pitfalls

### 5.3 神经科学类比规则
大部分直觉类比和例子，从脑科学 / 神经科学出发。

但每次类比必须固定有三个小节：
1. **这个类比哪里成立**
2. **这个类比哪里不成立**
3. **学生应该保留的直觉是什么**

禁止把“脑 = Transformer”写成字面同构。

你需要把论文分成三层：

### Tier A: 主叙事核心论文
这些论文要被真正写进主线，深入解释，不是只做 cite。

- Shannon
- Bengio 2003
- Sennrich BPE
- Transformer
- Adam / AdamW
- LayerNorm / pre-norm vs post-norm
- Kaplan / Chinchilla
- GPipe / Megatron / ZeRO
- PaLM
- FlashAttention / FlashAttention-2
- GPT-2 / GPT-3 / OPT / BLOOM / LLaMA / OLMo / DeepSeek
- InstructGPT / DPO / DeepSeekMath / DeepSeek-R1

### Tier B: 章节扩展论文
这些放到 section, box, case study, or further reading。

### Tier C: 2026 frontier delta
太新、太前沿、但与 2026 内容相关的论文，不要让它们破坏主线结构。可以放在：
- update cards
- appendix notes
- “what changed after the canonical recipe?” 小节
